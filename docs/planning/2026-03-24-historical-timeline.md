# 実装計画: 歴史タイムライン追加

## 要件

Globe.GL ビューアに紀元前1万年〜2023年の実測人口データを追加し、スライダーで全期間を操作できるようにする。

- **古代** (-10000〜1900): OWID 実測値。1000年刻み → 間を線形補間。色は全ドット暖色固定
- **近代** (1900〜2023): OWID 実測値。年刻み。色は暖色固定
- **未来** (2024〜2100): 現行の3シナリオモデル + コホート年齢色変え

## データの状態

- `owid_hist.json` に -10000〜2023 の全データあり（58,824点、168〜247カ国）
- 古代は1000年刻み、1800年代以降は10年刻み、1950年以降は年刻み
- 既に `pop_model_data.json` に 2000-2023 の hist フィールドとして追加済み

## 設計

### スライダー

現在: `<input type="range" min="2025" max="2100" step="1">`

3区間に分ける。**内部値は連続した整数**で、表示年に変換する:

| 区間 | 内部値 | 対応する年 | ステップ | 点数 |
|------|--------|-----------|---------|------|
| 古代 | 0〜11 | -10000〜0 (1000年刻み) | 1000年 | 12 |
| 近代 | 12〜135 | 1〜2023 (混合刻み) | 可変 | 124 |
| 未来 | 136〜212 | 2024〜2100 | 1年 | 77 |

→ 複雑すぎる。**もっとシンプルに:**

実際のデータがある年のリストを配列にして、スライダーの value = 配列インデックス。

```js
var TIMELINE = [-10000, -9000, ..., 0, 1000, ..., 1900, 1901, ..., 2023, 2024, ..., 2100];
// slider: min=0 max=TIMELINE.length-1
// 表示: TIMELINE[slider.value]
```

これで古代は自然にスキップが大きく、近現代は細かくなる。

### 人口比率の計算

- **基準年は各年の全ドット表示時の値**: ratio = pop(year) / pop(2023) のまま
  - ratio < 1 → 暗い/小さい、ratio > 1 → 明るい/大きい
  - 紀元前1万年はほぼ全ドット消灯（世界人口数百万人）→ 暗い地球
  - 近代に向かって光が点いていく → ドラマチック

- **古代〜近代**: `hist` の実測値から ratio を直接計算。補間なし（データがある年だけ）
- **未来**: 現行の `computeState()` でシナリオ別計算

### 色

- **-10000〜1949**: 全ドット暖色固定。コホート計算なし
- **1950〜2023**: 実測人口 + コホート逆算で年齢色変え（TFRデータが1950〜存在）
- **2024〜2100**: 現行のモデル予測 + コホート順算で年齢色変え

### UI 変更

- スライダーの左ラベル: "10000 BC"、右ラベル: "2100"
- 年表示: 負の年は "XXX BC" 表記
- シナリオボタンは未来区間でのみ有効（古代〜近代ではグレーアウト）
- 再生速度: 古代は1ステップで1000年進む、近代は1年ずつ — スライダー配列のインデックスを +1 するだけなので自然にそうなる

## 実装フェーズ

### Phase 1: データ準備
- `owid_hist.json` から全年の国別人口を抽出
- TIMELINE 配列を構築（データが存在する年のリスト + 2024-2100）
- `pop_history.json` として出力: `{code: {year: population, ...}, ...}`

### Phase 2: JS 側の updateYear 修正
- 年 <= 2023: `pop_history[code][year] / pop_history[code][2023]` で ratio 計算。色は暖色固定
- 年 > 2023: 現行の `computeState()` をそのまま使用
- TIMELINE 配列でスライダー↔年の変換

### Phase 3: UI 修正
- スライダーの min/max/step を TIMELINE インデックスベースに変更
- 年表示フォーマット（BC対応）
- シナリオボタンのグレーアウト（古代〜近代）

## リスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| pop_history.json のサイズ | LOW | 推定 200〜400KB。許容範囲 |
| 古代の ratio が極端に小さい (0.001 等) | MEDIUM | brightness の floor を調整して完全な暗闇にはしない |
| 再生がスカスカ（古代1000年刻みが一瞬で終わる） | LOW | 再生速度は配列インデックス +1 なので古代でもテンポ同じ |
| 補間なし → 古代で光がパッと変わる | LOW | 1000年刻みなのでむしろ自然 |

## 複雑度: 低〜中
